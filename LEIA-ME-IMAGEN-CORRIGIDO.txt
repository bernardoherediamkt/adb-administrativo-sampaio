ADB Administrativo — Correção do ADB Mídia-Designer para Imagen 4

O erro “models/imagen-4.0-fast-generate-001 is not found for API version v1, or is not supported for generateContent” acontecia porque o app estava chamando Imagen como se fosse Gemini Image/Nano Banana.

Agora a API faz dois modos:

1. Se GEMINI_IMAGE_MODEL começar com “imagen-”:
   - usa o endpoint correto do Imagen: /v1beta/models/...:predict
   - gera imagem por texto
   - ignora imagens enviadas como referência, porque Imagen 4 no Gemini API recebe texto como input

2. Se GEMINI_IMAGE_MODEL for “gemini-3.1-flash-image”, “gemini-3-pro-image” ou “gemini-2.5-flash-image”:
   - usa /v1/models/...:generateContent
   - aceita texto + imagens enviadas
   - só funcionará quando sua cota/billing desses modelos estiver liberada

Configuração recomendada na Vercel para funcionar agora com seu limite gratuito:

GEMINI_API_KEY = sua chave nova
GEMINI_IMAGE_MODEL = imagen-4.0-fast-generate-001

Se quiser melhor qualidade e também tiver limite disponível:

GEMINI_IMAGE_MODEL = imagen-4.0-generate-001

Depois de subir os arquivos no GitHub, faça Redeploy na Vercel.
